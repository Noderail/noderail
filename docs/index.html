What NodeRail Provides

Three capabilities that no content platform, wiki, or repository currently combines.

Permanent DOIs & Citation

Every node version receives a permanent Digital Object Identifier via Zenodo integration. Your work becomes a citable academic artifact — not a link that breaks.

Git-Based Lineage Tracking

Every contribution is timestamped and version-controlled. Fork, extend, or adopt any node while preserving a complete, auditable lineage back to the origin.

Structured Node Architecture

Fields, Frameworks, Measurements, and Projects are structured units — not freeform documents. They connect through citations and lineage, so progress compounds.

Four Node Types

Each node type is a structured, citable unit of intellectual work.

How NodeRail Works

Five steps from idea to citable, versioned knowledge infrastructure.

1

Create a Field

Define scope, constructs, ethics, and governance using the Field Constitution template.

2

Publish Nodes

Add Frameworks, Measurements, and Projects inside your Field using structured templates.

3

Version & Timestamp

Every change is recorded in Git with a permanent timestamp — your contribution is protected.

4

Issue a DOI

Each release receives a permanent DOI via Zenodo, making your node a citable academic artifact.

5

Fork & Continue

Others can fork, adopt, or extend your node — with full lineage preserved back to you.
